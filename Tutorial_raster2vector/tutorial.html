Extract data from figures using Inkscape

Last edited: 06/14/20

In this tutorial, we are going to extract data from figures in pdf’s using a vector graphics editor (Inkscape), a text editor, and a spreadsheet software. In principle, any vector graphics editor will work. As for the spreadsheet software, I am using LibreOffice Calc. Also, I have created this dummy pdf article which we will use as an example trough out this tutorial. The article has figures with graphs using either solid curves or scattered points.

Before we begin, we have to understand I little about digital graphic formats. Roughly speaking, we can divide digital images into two groups: vector or raster graphics. The main difference between them is that raster graphics (.png, .jpg, ...) are made of pixels, while vector graphics (.svg, .eps, ...) are made of vector Cartesian points. The figure below is an example of raster image. Note that, if you zoom in on the image, the image starts to get blurry.

Below, we have an example of a vector image. Note that, no matter how much you zoom in on the image there is no blur. This is what makes vector format perfect for high-quality publication figures or presentation on big screens. Raster images should be preferred mostly when the image is a photograph (sometimes, heatmaps are also better presented using raster format).

Firstly, I will show you how to extract data from vector images with solid lines. Them we will discuss how to do the same for figures with scattered points and for raster images.

We need to find the exact (x, y) coordinates of the first and the last data point. For that we can use Cross-multiplication to transform between Inkscape spacial coordinates to the “graph units”. From the gif below, we know that 11.964 mm is equivalent to 0.5, therefore, 18.246 is equivalent to 0.7625. This means that the y coordinate of the first data point is 0.7625.

If you do the same for the x coordinate of the first data point (see gif below), you will see that 12.706 mm is equivalent to 5, therefore, 2.532 is equivalent to 0.9964. This means that the x coordinate of the first data point is 0.9968.



We have to do the same for the last data point. You will see that the coordinates of the last data point are roughly (25, 0.0114).
